9.1. Код-ревью

/ Все изменения должны проходить через код-ревью.

/ Прежде чем отправлять код на ревью, проверь его:

  • Соблюдены ли правила форматирования?
  • Покрыты ли изменения тестами?
  • Достаточно ли понятно организован код?

9.2. Документирование кода

/ Добавляй комментарии только там, где логика сложна для понимания.

/ Вести документацию по API (Swagger, OpenAPI, Postman Collections).
