На главную

1. Общие принципы

1.1. Чистота кода

/ Пиши код так, чтобы он был понятен даже через полгода.

/ Названия переменных, функций, и файлов должны быть осмысленными и самодокументируемыми.

/ Каждый компонент или функция решает одну конкретную задачу.

1.2. Единообразие

/ Соблюдай стандарты форматирования, принятые в проекте (Prettier, ESLint).

/ Используй единый стиль для именования:

  • Файлы и папки в FSD: kebab-case
  • Файлы и папки в MODULE: PascalCase
  • Компоненты и классы: PascalCase
  • Переменные и функции: camelCase
  • Константы: camelCase

1.3. Четкое разделение ответственности

/ Не смешивай слои и модули. Компоненты не должны знать о сущностях

/ Разделять код на логические модули и не создавать монолитных файлов.

1.4. Осознанное программирование

/ Нельзя быть Monkey Developer — разработчиком, который бездумно копирует код, не понимая его смысла.

/ Нужно осознавать, зачем пишется каждая строка кода, а не просто повторять решения из интернета.

/ Развивать критическое мышление, анализировать архитектуру перед внесением изменений.

/ Учиться на ошибках, а не допускать их снова из-за отсутствия понимания причин.
